# Roadmap

## Generated Date

2026-05-27

## Vision

DocVault becomes a reliable, maintainable document management platform with a single, well-understood authentication system, a consolidated data model, a lean state management layer, and automated test coverage that catches regressions before they reach users.

## Out of Scope / Deferred

| Item | Reason |
|------|--------|
| Remove search pipeline abstraction immediately | The SearchOrchestrator → IndexManager → FallbackSearchProvider structure is overengineered, but it is not broken. It can be cleaned up as part of the real search index implementation (Long Term) rather than as a standalone refactor. |
| Connect analytics middleware to Mixpanel | No analytics requirement has been confirmed. The `analyticsTracker` middleware should be removed (Near Term cleanup), not completed. |
| Merge auth/user reducers as a standalone task | This is subsumed by the RTK migration (Mid Term). Doing a partial merge first would create churn. |

---

## Notes or Next Steps

- The Near Term section is sequenced by dependency: fix the auth bug first (it is the only blocker to testing the app at all), then remove the DEV_SKIP_AUTH default (so the fix is actually exercised), then address data integrity bugs.
- The Mid Term items (consolidate auth, add tests, resolve two-table architecture) are the "stability plateau" — after these, the codebase is safe to build features on.
- The Long Term items assume the platform is stable. Building batch operations or real search on the current foundation would be inadvisable.
