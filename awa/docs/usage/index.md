# Usage

This section covers how to use the AWA (Agentic Workflow Accelerator) in different ways and interact with its various components.

- **[CLI](/usage/cli)** - Command-line interface for managing services, running workflows, and system interaction
- **[API](/usage/api)** - REST API endpoints for programmatic access to AWA functionality (coming soon!)
- **[UI](/usage/ui)** - Web-based user interface for workflow management and monitoring (coming soon!)
- **[CI/CD](/usage/cicd)** - Containerized pipeline runner for executing workflows in GitHub Actions and other CI/CD environments
- **[MCP](/usage/mcp)** - Model Context Protocol server for integration with agentic tools
- **[A2A](/usage/a2a)** - Agent-to-Agent communication and interaction patterns (coming soon!)
- **Other Features**
  - **[Prompts and Templates](/usage/features/prompts-and-templates)** - Template management and prompt building capabilities
  - **[Agents](/usage/features/agents)** - Integration with various AI agents for workflow execution
  - **[OpenAI Agents SDK](/usage/features/openai-agents-sdk)** - Integration with OpenAI's Agents SDK
  - **[Agent Streaming](/usage/features/agent-streaming)** - Real-time streaming of agent execution and responses
  - **[File System Operations](/usage/features/file-system-operations)** - File and directory manipulation activities
  - **[Document Parsing](/usage/features/document-parsing)** - Document parsing activity for converting various document formats into markdown
  - **[Document Chunking](/usage/features/document-chunking)** - Document chunking with Chonkie for intelligent text segmentation
  - **[Logging](/usage/features/logging)** - Comprehensive logging system with component-based organization
  - **[Workflow Registration](/usage/features/workflow-registration)** - Dynamic workflow registration and discovery
